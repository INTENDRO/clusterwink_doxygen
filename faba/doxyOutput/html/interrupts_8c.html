ISR ( TIMER1_OVF_vect  )

shifting bufferOffset (Offset in Ringbuffer) and calling LED-Protocoll Interrupt

Timer is only activated in MODE_TWO_COLOR_FADER. There a color gradient is calculated with
"NUM_COLOR_GRADIENTS" color values. The colors are saved into the ring buffer.
Now, the attached modules are shifting in this ring buffer. At the begin, they are at the
one end of it and then shifting to the other end and back again.

[in]Timer1_OVF_vectimer overflow vector

ISR ( INT1_vect  )

sending LED Protocoll to RGBooster

The interrupt is forced the first time. Then it sets the byte of the first color to the output.
When the RGBooster finished sending the data it sets the interrupt pin to high and the interrupt fires again and sends the next byte (next color).
The interrupt function sends the same color data NUM_LEDS_PER_MODULE times (the same color within 1 module)
When finished all Bytes for each LED, the flag flag_sending_in_progress is set to false.

Parameters
[in]INT1_vectExternal Interrupt 1
Returns
no return value
